# Lead setup — one-time repo creation

**Who runs this:** one trainee per team, **appointed by the coach or by volunteering**. **Not the coach.** Each team creates its own repo. If you weren't appointed, skip this file — go to [README.md](README.md) and wait for your lead to share the new repo URL.

You'll do three things:

1. Download this source repo (the design system + guides)
2. Create a brand-new repo under your **@admiralsystems.io** GitHub account
3. Invite your team (teammates + coach) to it

When you're done, your team has its own private repo with the design system file ready to work on.

---

## Step 1 — Download this source repo

You're starting from the shared source repo that holds `base-design-system.pen` and these guides.

1. Open the source repo on GitHub.com in your browser
2. Click the green **Code** button → **Download ZIP**
3. Unzip it on your computer — you'll get a folder with `base-design-system.pen`, `README.md`, and `LEAD-SETUP.md`
4. Rename the folder to something team-specific, e.g. `admiral-pages-davis-team`

`[screenshot: GitHub.com Code → Download ZIP menu]`

---

## Step 2 — Create your team's new repo

1. Go to **GitHub.com** while signed in with your **@admiralsystems.io** account
2. Top-right **+** menu → **New repository**
3. Fill in:
   - **Owner:** your @admiralsystems.io account (not a personal account)
   - **Repository name:** match your folder name, e.g. `admiral-pages-davis-team`
   - **Private** ✅
   - **Initialize with README:** ❌ no (we'll bring our own)
   - **.gitignore / License:** None
4. Click **Create repository**
5. **Leave the page open** — you'll need it in the next step

`[screenshot: GitHub.com New Repository form]`

---

## Step 3 — Push your local folder into the new repo

1. Open **GitHub Desktop**
2. **File → Add Local Repository**
3. Pick the folder you unzipped in Step 1
4. GH Desktop will say "this folder is not a Git repository — create one" → click **create a repository**
5. Confirm name + path → **Create Repository**
6. You'll see all the files (`.pen`, `README.md`, `LEAD-SETUP.md`) listed in the left panel
7. **Summary** box: `Initial commit — design system + guides`
8. Click **Commit to main**
9. Top bar → **Publish repository** → pick **your new GitHub repo** as the destination → **Publish**

`[screenshot: Publish repository dialog]`

---

## Step 4 — Invite your team

Add your team members **and the coach** as collaborators.

1. On GitHub.com, open your new repo → **Settings → Collaborators and teams**
2. Click **Add people**
3. Add each person by their @admiralsystems.io GitHub username
4. **Role:** Write
5. Repeat for everyone in your team
